# Contributing

Thank you for improving the kit. Keep changes generic, small, and auditable.

1. Open an issue describing the reusable problem and affected layer: engine, theme, document structure, or consumer business content.
2. Fork the repository and branch from `main`.
3. Use only synthetic data and assets with a compatible license.
4. Run `python -m unittest discover -s tests -v` and the validation commands in the README.
5. Update tests and `CHANGELOG.md` for user-visible behavior.
6. Open a pull request using the template.

Do not contribute private brands, customer queries, internal identifiers, secrets, production URLs, screenshots from authenticated applications, copied proprietary skills, or business packages. Runtime changes must preserve consumer themes and business files during upgrades.

By submitting a contribution, you agree that it is licensed under Apache-2.0.
