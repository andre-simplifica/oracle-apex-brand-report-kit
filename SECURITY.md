# Security policy

## Supported versions

Security fixes are provided for the latest released minor version.

## Reporting a vulnerability

Do not open a public issue for credentials, authorization bypasses, data exposure, spreadsheet injection, or unsafe SQL. Use GitHub private vulnerability reporting for this repository. Include the affected version, impact, reproduction steps using synthetic data, and a proposed mitigation when available.

Do not include production data, tokens, cookies, private URLs, customer assets, database wallets, or screenshots from authenticated systems. Maintainers will acknowledge a complete report within seven days and coordinate disclosure after a fix is available.

## Consumer responsibility

This kit produces source templates. A consuming project must still apply its own authentication, authorization, organizational scope, secure credentials, code review, database locks, compilation controls, and deployment process. Never treat hidden buttons as authorization.
