# Agent-native workflow without Python

## Core rule

Use the agent's native file, browser, PDF, image, presentation, database, and spreadsheet capabilities. Python is not a prerequisite for extracting a visual identity, generating the consumer runtime, or running the result in Oracle APEX.

## Create a consumer

1. Read the consumer repository instructions and confirmed APEX project profile.
2. Inspect every relevant page, state, viewport, and original asset in the authorized visual source.
3. Create `brand-profile` and `report-profile` files that follow the bundled JSON schemas as normative contracts.
4. Render the bundled templates by replacing every required token, or create equivalent PL/SQL/CSS/JavaScript files that preserve the documented four-layer architecture.
5. Keep the engine generic, the theme consumer-owned, the document composition replaceable, and all queries and authorization in the existing domain package.
6. Present the complete file list and diffs before writing. Refuse unresolved placeholders, invented objects, secrets, private assets, and unsafe raw HTML/CSS values.
7. Configure one native Dynamic Content region with a short direct `RETURN CLOB` call. Do not introduce an iframe, external frontend server, CDN, or proprietary plug-in.
8. Compile only in the authorized DEV environment, inspect compilation errors, and validate authenticated runtime behavior.
9. Generate real portrait, landscape, and long-table PDFs, render every page with the available PDF tool, and inspect each page.
10. Record which deterministic scripts were not run. Never describe Python as an Oracle APEX or PL/SQL developer requirement.

## Optional deterministic path

When Python 3.10+ is already available, the bundled scripts can enforce schemas, generate a repeatable scaffold and manifest, detect update conflicts, scan sensitive data, run tests, and package the skill. Their absence changes the validation evidence, not the APEX runtime architecture.

For upgrades without the scripts, compare the installed manifest and checksums with the new tagged engine templates, preserve all theme-managed and project-owned files, review each diff, and retain a repository or equivalent rollback path.
